Overview

Juxy is a unit testing library for XSLT written in Java. It provides a way to start any XSLT transformation from Java within specified context, to obtain the result of the transformation (document fragment) and to apply various checks on the result.

Juxy not only allows you to start transformation, but also provides a way to configure transformation context: global variables, parameters, current node and so on. The transformation can be started by calling or applying templates. For called template you can specify name of the template which will be called as well as template parameters which should be passed to the template. For applied templates you can specify select expression, mode and template parameters.

For now Juxy provides two ways of the result checking:

  • you can evaluate XPath expression over the resulting Node (requires Jaxen to be added in the classpath)
  • or you can compare resulting fragment with expected XML
Or you can use XMLUnit for the result checking.

Juxy does not use any XSLT processor specific extensions, it should work with all TRaX compliant XSLT processors. For now it is known to work with Apache Xalan and Saxon (both Saxon 6.5.X and Saxon 8.X are supported). XSLT 2.0 templates are known to work with Saxon 8.X, although they are not thoroughly tested.

How it works

Juxy works by dynamically creating XSLT stylesheet. From that stylesheet the tested stylesheet is imported via xsl:import instruction, thus all its templates and variables got lower priority. Any variables and parameters specified in the transformation context will be placed in the generated stylesheet. If current node was specified, then corresponding xsl:for-each is added to that stylesheet.

The returned result Node is always DocumentFragment node. It allows to return not well-formed documents containing, for example, only text, or more than one root node. This however could affect XPath expressions which are used to check the result.
